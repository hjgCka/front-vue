### 常用类型

#### any类型

声明为any的话，会关闭类型检查。

没有声明类型的话，会进行自动推断，如果无法推断出类型就会认为是any。

有`noImplicitAny`编译选项，如果自动推断出any就会报错。

```typescript
let b;  //不写类型，推断为any类型
b = 99;
b = false;
```

由于可以将`any`类型变量赋值给`let s : string`的变量，所以这会污染其它类型的变量。



#### unknown

这个类型相当于安全的any类型，凡是需要any类型的地方，通常都应该优先考虑设为`unknown`类型。

这个类型无法直接赋值，但是可以通过缩小类型，进行安全的赋值：

```typescript
let a : unknown
a = 99;

let n : number;
if (typeof a === 'number') {
    n = a;
}

// n = a as number
// n = <number>a
```

可参考阮一峰的ts教程：类型断言章节。



#### never

空类型。不可赋值给它任何值，否则报错。



#### 元组

比如数组有更强的约束，比如长度固定，类型固定。



#### type命令

通过type命令来定义类型的别名。



#### void类型

一般用来作为函数的返回值，如果返回值类型为void，那么就不允许使用return。

但是void 类型允许返回`undefined`或`null`。

JavaScript 规定，如果函数没有返回值，就等同于返回`undefined`。

可参考阮一峰的：TypeScript 的函数类型。



#### class

JavaScript 的类本质上是一个构造函数



#### type命令

type用来定义一个类型的别名，有时可以简化代码量。
```typescript
type Age : number;
let age: Age = 15;
```



#### 字面量类型（值类型）

单个值也是一种类型，被称为“值类型”。

```typescript
let x : ‘hello';
x = 'hello'; //正确
x = ‘word'; //错误
```



#### 包装类型

比如String, Number, Boolean这样的构造函数。

对内存不友好，而且ts官方内置方法很多的参数是小写的类型，所以尽量用小写。

尽量使用string, number等小写类型，而不是大写类型String, Number。后者返回的是包装类型对象，而不是普通的原始类型。



#### 类与接口

多个接口之间会进行合并。

ts不允许2个同名的class，但是如果一个class和一个interface同名，那么接口就会被合并进类。
